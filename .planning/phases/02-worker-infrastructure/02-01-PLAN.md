Create the WorkerManager class that implements the three-tier worker fallback strategy (SharedWorker → DedicatedWorker → main thread) with optional storage persistence available via explicit persistStorage().

Purpose: This is the foundation of LokulMem's execution model. Heavy embedding operations must run off the main thread for performance, but we need graceful degradation when workers aren't available.

Output: WorkerManager.ts with full fallback chain, Persistence.ts for storage persistence, and core types.
</objective>

<task type="auto">
  <name>Task 3: Create WorkerManager with fallback chain</name>
  <files>src/core/WorkerManager.ts</files>
  <action>
    Create src/core/WorkerManager.ts implementing the three-tier fallback:

    1. Import types from ./types.js and requestPersistence from ./Persistence.js

    2. Export class WorkerManager:
       - private port: PortLike | null = null
       - private worker: SharedWorker | Worker | null = null
       - private workerType: WorkerType = 'main-thread'
       - private config: WorkerConfig
       - private persistenceStatus: PersistenceStatus | null = null
       - private isReady = false
       - private client: WorkerClient | null = null

    3. Method async initialize(config: WorkerConfig, onProgress?: ProgressCallback): Promise<void>
       - Store config
       - Call doInitialize(config) to get PortLike (implements fallback chain)
       - Create WorkerClient with the port
       - Set up progress handling via onProgress callback
       - Send INIT request and wait for completion
       - Set isReady = true
       - Return void

    4. Private async doInitialize(config: WorkerConfig): Promise<PortLike>
       - If workerType is 'auto' or 'shared': try SharedWorker
         - new SharedWorker(config.workerUrl, { type: 'module', name: 'lokulmem-v1' })
         - Call port.start()
         - On success: store port, set workerType='shared', return port as PortLike
         - On failure: if config.workerType==='shared', throw; else fall through
       - If workerType is 'auto' or 'dedicated': try DedicatedWorker
         - new Worker(config.workerUrl, { type: 'module' })
         - On success: store worker, wrap in PortLike adapter (delegates to worker.postMessage/worker.onmessage), set workerType='dedicated', return PortLike
         - On failure: if config.workerType==='dedicated', throw; else fall through
       - Main thread fallback: createMainThreadPort(), set workerType='main-thread', return port
       - NOTE: Persistence is NOT auto-called; user must call persistStorage() explicitly per Phase 2 decisions

    Also add ProgressCallback type: (stage: InitStage, stageProgress: number, overallProgress: number, message?: string) => void

    5. Method getWorkerType(): WorkerType - returns current workerType

    6. Method getPersistenceStatus(): PersistenceStatus | null

    7. Method isInitialized(): boolean - returns isReady

    8. Method getClient(): WorkerClient | null - returns client if initialized

    9. Method terminate(): void - clean up worker/port

    10. Private createDedicatedWorkerPort(worker: Worker): PortLike - wraps Worker in PortLike adapter:
       - Returns object with postMessage: (data, transfer) => worker.postMessage(data, transfer)
       - onmessage getter/setter delegates to worker.onmessage
       - onmessageerror getter/setter delegates to worker.onmessageerror
       - close() calls worker.terminate()

    11. Method async persistStorage(): Promise<PersistenceStatus> - explicit persistence call per Phase 2 decisions:
       - Call requestPersistence() and store result
       - Return persistence status
       - This is called explicitly by user, NOT during initialize()

    12. Private createMainThreadPort(): PortLike - placeholder for main thread execution
       (Placeholder that will be wired to in-memory worker handlers in Plan 03)

    Per RESEARCH.md patterns:
    - Use try/catch for feature detection (not UA sniffing)
    - Call port.start() for SharedWorker
    - DedicatedWorker is MessagePort-like (use directly)
  </action>
