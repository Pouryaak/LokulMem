Purpose: This is the foundation of LokulMem's execution model. Heavy embedding operations must run off the main thread for performance, but we need graceful degradation when workers aren't available.

Output: WorkerManager.ts with full fallback chain, Persistence.ts for storage persistence, and core types.

<task type="auto">
  <name>Task 3: Create WorkerManager with fallback chain</name>
  <files>src/core/WorkerManager.ts</files>
  <action>
    Create src/core/WorkerManager.ts implementing the three-tier fallback:

    1. Import types from ./types.js and requestPersistence from ./Persistence.js

    2. Export class WorkerManager:
       - private port: MessagePort | null = null
       - private worker: SharedWorker | Worker | null = null
       - private workerType: WorkerType = 'main'
       - private config: WorkerConfig
       - private persistenceStatus: PersistenceStatus | null = null

    3. Method async initialize(config: WorkerConfig): Promise<MessagePort>
       - Store config
       - Call requestPersistence() FIRST (before any worker spawn)
       - Store persistence result
       - If workerType is 'auto' or 'shared': try SharedWorker
         - new SharedWorker(config.workerUrl, { type: 'module', name: 'lokulmem-v1' })
         - Call port.start()
         - On success: store port, set workerType='shared', return port
         - On failure: if config.workerType==='shared', throw; else fall through
       - If workerType is 'auto' or 'dedicated': try DedicatedWorker
         - new Worker(config.workerUrl, { type: 'module' })
         - On success: store worker, port=worker (as MessagePort-like), set workerType='dedicated', return port
         - On failure: if config.workerType==='dedicated', throw; else fall through
       - Main thread fallback: createMainThreadPort(), set workerType='main', return port

    4. Method getWorkerType(): WorkerType - returns current workerType

    5. Method getPersistenceStatus(): PersistenceStatus | null

    6. Method terminate(): void - clean up worker/port

    7. Private createMainThreadPort(): MessagePort - creates a MessagePort-like interface for main thread execution
       (This will be a stub that routes to in-memory handlers - full implementation in next plan)

    Per RESEARCH.md patterns:
    - Use try/catch for feature detection (not UA sniffing)
    - Call port.start() for SharedWorker
    - DedicatedWorker is MessagePort-like (use directly)
  </action>
