<task type="auto">
  <name>Task 2: Create EmbeddingEngine class with CDN and airgap support</name>
  <files>src/worker/EmbeddingEngine.ts</files>
  <action>
    Create src/worker/EmbeddingEngine.ts with:

    1. EmbeddingConfig interface extending InitPayload modelConfig:
       - modelName: string (default: 'Xenova/all-MiniLM-L6-v2')
       - localModelBaseUrl?: string (triggers airgap mode)
       - embeddingDims: number (default: 384 for MiniLM)
       - onnxPaths?: string | Record<string, string> (custom WASM paths)
       - cacheSize?: number (default: 1000)
       - enableCache?: boolean (default: true)

    2. EmbeddingEngine class as singleton:
       - private pipeline: FeatureExtractionPipeline | null
       - private config: EmbeddingConfig
       - private initialized: boolean
       - private modelName: string (stored for DB metadata)

    3. initialize(config: EmbeddingConfig, onProgress: (stage: string, progress: number) => void): Promise<void>
       - Import { env } from '@huggingface/transformers' (NOT @xenova/transformers)
       - If localModelBaseUrl provided: airgap mode
         - env.allowLocalModels = true
         - env.allowRemoteModels = false
         - env.localModelPath = localModelBaseUrl
         - Validate localModelBaseUrl ends with proper path separator
       - Else: CDN mode
         - env.allowLocalModels = false
         - env.allowRemoteModels = true
         - env.useBrowserCache = true (explicitly enable Cache API)
       - Progress reporting: "Downloading tokenizer" (0-30%), "Loading model weights" (30-80%), "Compiling WASM" (80-100%)
       - Use pipeline('feature-extraction', modelName, { dtype: 'q8', progress_callback: ... }) - NOT quantized: true (old API)
       - Store modelName and embeddingDims in engine state

    4. embed(text: string): Promise<Float32Array>
       - Throws if not initialized
       - Returns Float32Array of embedding dimensions
       - Use pipeline with pooling: 'mean', normalize: true
       - Direct call: await this.pipeline!(text, { pooling: 'mean', normalize: true })

    5. embedBatch(texts: string[]): Promise<Float32Array[]>
       - Batch embedding with max 32 items (per CONTEXT.md decision)
       - If texts.length > 32, internally chunk into batches of 32 and process sequentially
       - Return concatenated results in original order

    6. getModelInfo(): { modelName: string, embeddingDims: number, initialized: boolean }
       - Returns current model metadata for DB storage

    7. Error handling:
       - ModelLoadError: clear recovery hints (check localModelBaseUrl, network connectivity)
       - DimensionMismatchError: thrown if embed() returns unexpected dimensions

    Use proper TypeScript types from @huggingface/transformers. Import type { FeatureExtractionPipeline } from '@huggingface/transformers'.
  </action>
  <verify>npm run typecheck passes with no errors</verify>
  <done>EmbeddingEngine.ts exists with initialize(), embed(), embedBatch(), getModelInfo() methods using @huggingface/transformers</done>
