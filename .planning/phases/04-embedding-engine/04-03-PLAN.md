<task type="auto">
  <name>Task 2: Configure Vite to bundle ORT WASM assets</name>
  <files>vite.config.ts</files>
  <action>
    Update vite.config.ts:

    1. Import copy plugin:
       import copy from 'rollup-plugin-copy'

    2. Add copy plugin to plugins array:
       ```typescript
       copy({
         targets: [
           {
             src: 'node_modules/onnxruntime-web/dist/*.wasm',
             dest: 'dist',
           },
           {
             src: 'node_modules/onnxruntime-web/dist/ort-wasm*.mjs',
             dest: 'dist',
           },
         ],
         hook: 'writeBundle', // Copy after bundle is written
       })
       ```

    3. Add publicDir configuration to serve WASM files in dev mode:
       publicDir: 'public',

    4. Create public/ folder with .gitkeep if it doesn't exist

    5. Add asset handling for .wasm files in build.rollupOptions:
       ```typescript
       rollupOptions: {
         output: {
           assetFileNames: (assetInfo) => {
             if (assetInfo.name?.endsWith('.wasm')) {
               return '[name][extname]'
             }
             return 'assets/[name]-[hash][extname]'
           },
         },
       },
       ```

    Note: onnxruntime-web is a transitive dependency via @xenova/transformers. The WASM files need to be available at runtime for the embedding engine to work.

    Alternative approach if copy plugin has issues: Use vite-plugin-static-copy which is more actively maintained.
  </action>
  <verify>npm run build completes without errors</verify>
  <done>vite.config.ts has copy plugin configuration for WASM assets</done>
</task>

    2. In initialize(), configure ONNX WASM paths before loading model:
       ```typescript
       import { env } from '@xenova/transformers';

       // Configure ONNX WASM paths if provided
       if (config.onnxPaths) {
         if (typeof config.onnxPaths === 'string') {
           // Base URL for all WASM files
           env.backends.onnx.wasm.wasmPaths = config.onnxPaths;
         } else {
           // Individual file paths
           env.backends.onnx.wasm.wasmPaths = config.onnxPaths;
         }
       }
       ```

       ```typescript
       }
       ```
