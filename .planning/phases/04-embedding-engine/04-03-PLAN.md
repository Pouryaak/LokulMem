<task type="auto">
  <name>Task 2: Configure Vite to bundle ORT WASM assets for dev and production</name>
  <files>vite.config.ts</files>
  <action>
    Update vite.config.ts:

    1. Import copy plugin:
       import { viteStaticCopy } from 'vite-plugin-static-copy'

    2. Add copy plugin to plugins array:
       ```typescript
       viteStaticCopy({
         targets: [
           {
             src: 'node_modules/onnxruntime-web/dist/*.wasm',
             dest: '.',
           },
           {
             src: 'node_modules/onnxruntime-web/dist/ort-wasm*.mjs',
             dest: '.',
           },
         ],
       })
       ```

    3. Ensure publicDir is configured (default is 'public'):
       publicDir: 'public',

    4. Create public/ folder with .gitkeep if it doesn't exist

    5. The plugin will:
       - Copy WASM files to public/ during dev (so Vite dev server serves them)
       - Copy WASM files to dist/ during build

    6. Add asset handling for .wasm files in build.rollupOptions:
       ```typescript
       rollupOptions: {
         output: {
           assetFileNames: (assetInfo) => {
             if (assetInfo.name?.endsWith('.wasm')) {
               return '[name][extname]'
             }
             return 'assets/[name]-[hash][extname]'
           },
         },
       },
       ```

    Note: onnxruntime-web is a transitive dependency via @huggingface/transformers. The WASM files need to be available at runtime for the embedding engine to work in both dev and production.

    Alternative: If vite-plugin-static-copy has issues, copy files manually in a prebuild script or use rollup-plugin-copy with writeBundle hook plus manual copy to public/ for dev.
  </action>
  <verify>npm run build completes without errors</verify>
  <done>vite.config.ts has vite-plugin-static-copy configuration for WASM assets</done>
</task>

       ```typescript
         localModelBaseUrl?: string;
         embeddingDims: number;
         onnxPaths?: string | Record<string, string>;
         cacheSize?: number;
         enableCache?: boolean;
       }
       ```

    2. In initialize(), configure ONNX WASM paths BEFORE loading model:
       ```typescript
       import { env } from '@huggingface/transformers';

       // Configure ONNX WASM paths if provided
       if (config.onnxPaths) {
         if (typeof config.onnxPaths === 'string') {
           // Base URL for all WASM files
           env.backends.onnx.wasm.wasmPaths = config.onnxPaths;
         } else {
           // Individual file paths
           env.backends.onnx.wasm.wasmPaths = config.onnxPaths;
         }
       }

       // If onnxPaths not provided but localModelBaseUrl is provided (airgap mode),
       // default WASM paths to localModelBaseUrl
       if (!config.onnxPaths && config.localModelBaseUrl) {
         env.backends.onnx.wasm.wasmPaths = config.localModelBaseUrl;
       }
       ```

    3. For CDN mode, WASM files should be loaded from the default location or the served public/dist directory. The vite-plugin-static-copy ensures they're available at root.

    This ensures WASM binaries are loaded from the correct location in both CDN and airgap modes.
