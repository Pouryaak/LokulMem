         embedding: Float32Array;
         timestamp: number; // Unix timestamp for age calculation

    2. CacheStats interface:
       interface CacheStats {
         size: number;
         maxSize: number;
         hitCount: number;
         missCount: number;
         hitRate: number; // 0-1
         oldestEntryAgeMs: number; // Age of oldest entry in cache
         memoryEstimateBytes: number; // Estimated memory usage
       }

    3. LRUCache class:
       - private cache: Map<string, CacheEntry>
       - private maxSize: number
       - private stats: { hits: number; misses: number }
       - private dims: number // Parameterized embedding dimensions (default: 384)
       - private readonly BYTES_PER_FLOAT = 4

       constructor(maxSize: number = 1000, dims: number = 384)
       - Store dims for memory calculation

       get(key: string): Float32Array | undefined
       - Increment stats.hits on hit, stats.misses on miss
       - Update entry.accessCount and timestamp on hit
       - Move accessed entry to end of Map (most recent)

       set(key: string, embedding: Float32Array): void
       - If key exists, update and move to end
       - If new key and cache at capacity, delete first entry (LRU)
       - Store with timestamp and accessCount=1

       has(key: string): boolean

       delete(key: string): boolean

       clear(): void

       getStats(): CacheStats
       - Calculate hitRate = hits / (hits + misses) or 0
       - Find oldest entry by timestamp
       - memoryEstimateBytes = size * this.dims * BYTES_PER_FLOAT (uses parameterized dims)

       getSize(): number

    4. Export LRUCache and CacheStats

    Use Map to preserve insertion order for LRU semantics. The first entry is oldest, last is newest.

    IMPORTANT: dims is now a constructor parameter, not hardcoded to 384. This allows different models with different embedding dimensions.

    Add PromiseQueue class to src/worker/LRUCache.ts (or create separate file if preferred):

    1. QueueItem interface:
       interface QueueItem<T> {
         fn: () => Promise<T>;
         resolve: (value: T) => void;
         reject: (error: Error) => void;
       }

    2. PromiseQueue class:
       - private queue: QueueItem<unknown>[]
       - private running: boolean

       add<T>(fn: () => Promise<T>): Promise<T>
       - Create promise that resolves/rejects when fn completes
       - Add to queue
       - Trigger processQueue()
       - Return promise

       private processQueue(): void
       - If running, return (already processing)
       - If queue empty, return
       - Set running = true
       - Dequeue first item
       - Execute fn(), then resolve/reject promise
       - Set running = false
       - Call processQueue() recursively for next item

       size(): number
       - Return queue length

    This ensures only one embedding operation runs at a time, preventing race conditions and memory pressure.
  </action>

       - Create PromiseQueue

    4. Update embed() to use cache and queue:
       ```typescript
       async embed(text: string): Promise<Float32Array> {
         if (!this.initialized) throw new Error('Not initialized');
         if (!this.pipeline) throw new Error('No pipeline');

         // Check cache first (outside queue for speed)
         if (this.cacheEnabled) {
           const cached = this.cache.get(text);
           if (cached) return cached;
         }

         // Use queue to prevent concurrent calls
         return this.queue.add(async () => {
           // Double-check cache after queue (another call may have populated it)
           if (this.cacheEnabled) {
             const cached = this.cache.get(text);
             if (cached) return cached;
           }

           // Compute embedding - direct pipeline call
           const result = await this.pipeline!(text, { pooling: 'mean', normalize: true });
           const embedding = result.data as Float32Array;

           // Store in cache
           if (this.cacheEnabled) {
             this.cache.set(text, embedding);
           }

           return embedding;
         });
       }
       ```

    5. Update embedBatch():
       - Use queue.add() for the entire batch
       - Check cache for each text individually
       - Only compute embeddings for cache misses
       - Store computed embeddings in cache
       - Return results in original order
       - Internally chunk to max 32 items per batch (consistent with embedBatch behavior)

    6. Add getCacheStats(): CacheStats method:
       - Returns this.cache.getStats()
       - Throws if cache not enabled

    7. Memory warning (per CONTEXT.md):
       - In getCacheStats(), if memoryEstimateBytes > 10MB, log warning
       - If > 50MB, log critical warning
       - These are warnings only (no auto-shrink by default)

    The cache key is the raw text string. This is simple and effective for exact matches.
