<task type="auto">
  <name>Task 1: Create MemoryRepository with CRUD operations</name>
  <files>src/storage/MemoryRepository.ts</files>
  <action>
    Create src/storage/MemoryRepository.ts:

    1. Imports:
       - LokulDatabase from './Database.js'
       - MemoryInternal from '../internal/types.js'
       - MemoryType, MemoryStatus from '../types/memory.js'
       - memoryToDb, memoryFromDb, DbMemoryRow from './embeddingStorage.js'

    2. Define MemoryFilter interface for query filtering:
       ```typescript
       interface MemoryFilter {
         types?: MemoryType[];
         status?: MemoryStatus;
         clusterId?: string;
         pinned?: boolean;  // Filter interface uses boolean for DX
         minStrength?: number;
       }
       ```

       Note: Repository converts pinned boolean to pinnedInt (0|1) for DB queries.

    3. MemoryRepository class:
       - constructor(private db: LokulDatabase) {}

    4. Core CRUD methods:
       - async create(memory: MemoryInternal): Promise<void>
         - Convert to DbMemoryRow using memoryToDb
         - Use this.db.memories.add()

       - async getById(id: string): Promise<MemoryInternal | null>
         - Query by primary key
         - Convert back using memoryFromDb if found

       - async update(memory: MemoryInternal): Promise<void>
         - Convert to DbMemoryRow
         - Use this.db.memories.put() (upsert behavior)

       - async delete(id: string): Promise<void>
         - Use this.db.memories.delete()

    5. Query methods using indexes:
       - async findByTypeAndStatus(type: MemoryType, status: MemoryStatus): Promise<MemoryInternal[]>
         - Use multiEntry types index: where('types').equals(type)
         - Then filter results by status in JavaScript (no compound index possible with multiEntry)
         - For ≤3000 memories, this is efficient enough

       - async findActiveByRecency(limit: number): Promise<MemoryInternal[]>
         - Use compound index [status+lastAccessedAt]
         - between(['active', 0], ['active', Date.now()])
         - reverse().limit(limit)

       - async findByCluster(clusterId: string, status?: MemoryStatus): Promise<MemoryInternal[]>
         - If status provided: use [clusterId+status] compound index
         - Else: use clusterId index

       - async findPinned(): Promise<MemoryInternal[]>
         - Use pinnedInt index with equals(1) (pinnedInt is number 0|1, not boolean)

       - async findByMentionCount(minCount: number, limit: number): Promise<MemoryInternal[]>
         - Use mentionCount index with above(minCount)
         - reverse().limit(limit)

       - async findByBaseStrengthRange(min: number, max: number, status?: MemoryStatus): Promise<MemoryInternal[]>
         - If status: use [status+baseStrength] compound index with between([status, min], [status, max])
         - Else: use baseStrength index with between(min, max)

    6. Bulk operations:
       - async bulkCreate(memories: MemoryInternal[]): Promise<void>
         - Map to DbMemoryRow[] using memoryToDb
         - Use this.db.memories.bulkAdd()

       - async bulkUpdate(memories: MemoryInternal[]): Promise<void>
         - Use this.db.memories.bulkPut()

    7. Count methods:
       - async count(filter?: MemoryFilter): Promise<number>
         - Apply filters and return count

    8. Utility methods:
       - async exists(id: string): Promise<boolean>
       - async getAllIds(): Promise<string[]>

    All query methods must convert DbMemoryRow results back to MemoryInternal using memoryFromDb.
    Handle null/undefined embeddings gracefully (should not happen but defensive coding).
  </action>
