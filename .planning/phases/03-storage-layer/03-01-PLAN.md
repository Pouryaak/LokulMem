<task type="auto">
  <name>Task 2: Create embedding storage conversion utilities</name>
  <files>src/storage/embeddingStorage.ts</files>
  <action>
    Create src/storage/embeddingStorage.ts with:

    1. Define EXPECTED_EMBEDDING_DIM = 384 (MiniLM-L6-v2 dimension)

    2. Export toDbFormat(embedding: Float32Array): ArrayBuffer
       - Use explicit slice: embedding.buffer.slice(embedding.byteOffset, embedding.byteOffset + embedding.byteLength)
       - This avoids the TypedArray view footgun where the underlying buffer may be larger

    3. Export fromDbFormat(buffer: ArrayBuffer): Float32Array
       - Create new Float32Array from buffer
       - Validate length === EXPECTED_EMBEDDING_DIM
       - Throw descriptive error if dimension mismatch

    4. Export interface DbMemoryRow that represents the database row format:
       - Same as MemoryInternal but with embeddingBytes: ArrayBuffer instead of embedding: Float32Array

    5. Export memoryToDb(memory: MemoryInternal): DbMemoryRow
       - Convert embedding to ArrayBuffer using toDbFormat
       - Return object with all MemoryInternal fields except embedding replaced with embeddingBytes

    6. Export memoryFromDb(row: DbMemoryRow): MemoryInternal
       - Convert embeddingBytes back to Float32Array using fromDbFormat
       - Return full MemoryInternal object

    Add JSDoc comments explaining why the explicit slice is necessary.
  </action>
  <verify>npm run typecheck passes; write a quick test to verify round-trip conversion works</verify>
  <done>embeddingStorage.ts exists with toDbFormat, fromDbFormat, memoryToDb, memoryFromDb functions and dimension validation</done>
</task>
